Conclusion: Solving the Netflix Content Classification Problem

1. The Core Problem

Our objective was to automate the categorization of Netflix content into Movies and TV Shows. Challenges included noisy metadata, missing fields (director/cast) and an imbalanced class distribution (~70% Movies, ~30% TV Shows).

2. The Solution & Pipeline

  • Data Preparation: fill or impute missing values and extract clean sub-features (duration → minutes/seasons, date parts).
  • Feature Engineering: transform text columns like listed_in into compact signals (genre counts), drop noisy text.
  • Model Experimentation: trained and compared 6 algorithms (Logistic Regression, Naive Bayes, SVM, Random Forest, Gradient Boosting, MLP).

3. Final Choice

After comparing Accuracy, F1, ROC-AUC and confusion matrices, the Random Forest Classifier was chosen as the production model due to high accuracy, robustness, and low inference cost.

Key takeaways

  • Final model: Random Forest
  • Peak accuracy: ~99%
  • Class balance: ~70% Movies / 30% TV Shows
  • Production: fast inference (ms) and robust to missing metadata
Final verdict: Random Forest pipeline — scalable & reliable
